# 🌱 总目标

> **JWT + Spring Security Filter**
>
> - JWT **不放权限**
> - 权限 **放 Redis**
> - 有 **短期 Token（Access）**
> - 有 **长期 Token（Refresh）**

并且你要**理解每一步为什么这么做**。

# 🧠 先给你一张“唯一正确”的全景图（请慢慢看）

```
【登录】
账号 + 密码
   ↓
Spring Security 认证
   ↓
生成 Access Token（15 分钟）
生成 Refresh Token（7 天）
   ↓
把「权限」存 Redis
把「token 会话」存 Redis
   ↓
返回给前端

【访问接口】
请求 + Access Token
   ↓
Security Filter
   ↓
解析 JWT（只拿 userId / tokenId）
   ↓
Redis 校验 token 是否有效
Redis 取权限
   ↓
SecurityContext
   ↓
@PreAuthorize 生效
```

👉 **你只要记住这条线，别想别的**

# 第一部分：为什么要“短期 + 长期 Token”（先搞明白）

## ❌ 只有一个 JWT 会怎样？

- JWT 过期长 → **被盗风险大**
- JWT 过期短 → **用户体验差**

## ✅ 解决方案：一短一长

| Token         | 干什么                        |
| ------------- | ----------------------------- |
| Access Token  | 每次请求用，**安全**          |
| Refresh Token | Access 过期时换新的，**体验** |

> **Access 被盗，15 分钟自动失效**
> **Refresh 被盗，也能在 Redis 里踢掉**

------

# 第二部分：JWT 里到底放什么？（别再纠结）

## ❌ 永远不要放

- 权限
- 角色
- 业务信息

## ✅ 只放这 4 个

```json
{
  "uid": 2,
  "sub": "12018007",
  "jti": "uuid",
  "type": "ACCESS"
}
```

| 字段 | 为什么                |
| ---- | --------------------- |
| uid  | Redis 查权限用        |
| jti  | 踢人、失效            |
| type | 区分 Access / Refresh |
| sub  | 人类可读              |

👉 **JWT 只证明“你是谁”，不证明“你能干嘛”**

------

# 第三部分：Redis 是“真正的权限与会话中心”

## Redis 里存什么？（记住这 4 个 Key）

```text
login:access:{jti}   → userId   （15 分钟）
login:refresh:{jti}  → userId   （7 天）
login:perms:{userId} → 权限集合 （7 天）
login:online:{userId}→ refreshJti
```

### 为什么这样设计？

| 需求         | Redis 怎么支持 |
| ------------ | -------------- |
| Access 失效  | TTL            |
| Refresh 失效 | TTL            |
| 踢人         | 删 refresh     |
| 改权限       | 删 perms       |

```java
    ACCESS, REFRESH
}
```

## Step 2️⃣ JwtTokenService（只做两件事）

```java
@Service
public class JwtTokenService {

    private static final String SECRET = "wsw-secret-key";

    private static final long ACCESS_EXPIRE = 15 * 60 * 1000;
    private static final long REFRESH_EXPIRE = 7 * 24 * 60 * 60 * 1000;

    public String createToken(Long userId, String username, TokenType type) {
        long expire = type == TokenType.ACCESS ? ACCESS_EXPIRE : REFRESH_EXPIRE;

        return Jwts.builder()
            .setId(UUID.randomUUID().toString())
            .setSubject(username)
            .claim("uid", userId)
            .claim("type", type.name())
            .setExpiration(new Date(System.currentTimeMillis() + expire))
            .signWith(SignatureAlgorithm.HS512, SECRET)
            .compact();
    }

    public Claims parseToken(String token) {
        return Jwts.parser()
            .setSigningKey(SECRET)
            .parseClaimsJws(token)
            .getBody();
    }
}
```

👉 **这里绝对不碰 Redis**

------

## Step 3️⃣ 登录成功时（真正的“会话创建”）

```java
public LoginResult login(LoginCommand cmd) {

    // ① 账号密码认证（省略）
    Long userId = loginUser.getUser().getUserId();

    // ② 生成 Token
    String accessToken = jwtTokenService.createToken(
        userId, username, TokenType.ACCESS
    );
    String refreshToken = jwtTokenService.createToken(
        userId, username, TokenType.REFRESH
    );

    Claims a = jwtTokenService.parseToken(accessToken);
    Claims r = jwtTokenService.parseToken(refreshToken);

    // ③ Redis：Access
    redis.set("login:access:" + a.getId(), userId, 15min);

    // ④ Redis：Refresh
    redis.set("login:refresh:" + r.getId(), userId, 7days);

    // ⑤ Redis：权限
    redis.sadd("login:perms:" + userId, permissions);
    redis.expire("login:perms:" + userId, 7days);

    // ⑥ 单点登录
    redis.set("login:online:" + userId, r.getId(), 7days);

    return new LoginResult(accessToken, refreshToken);
}
```

💡 **这一步是“登录 = 创建会话”**

## Step 4️⃣ JwtAuthenticationFilter（只处理 Access）

```java
protected void doFilterInternal(...) {

    String token = resolveToken(request);
    if (token == null) {
        chain.doFilter(...);
        return;
    }

    Claims claims = jwtTokenService.parseToken(token);

    // ① 只允许 Access
    if (!"ACCESS".equals(claims.get("type"))) {
        chain.doFilter(...);
        return;
    }

    String jti = claims.getId();
    Long userId = claims.get("uid", Long.class);

    // ② Redis 校验 Access
    if (!redis.exists("login:access:" + jti)) {
        chain.doFilter(...);
        return;
    }

    // ③ Redis 取权限
    Set<String> perms = redis.smembers("login:perms:" + userId);

    // ④ 构造 Authentication
    Authentication auth =
        new UsernamePasswordAuthenticationToken(
            userId,
            null,
            permsToAuthorities(perms)
        );

    SecurityContextHolder.getContext().setAuthentication(auth);
    chain.doFilter(...);
}
```

👉 **Filter = 请求时恢复会话**

## Step 5️⃣ 刷新 Token（不走 Filter）

```java
@PostMapping("/auth/refresh")
public LoginResult refresh(String refreshToken) {

    Claims claims = jwtTokenService.parseToken(refreshToken);

    if (!"REFRESH".equals(claims.get("type"))) {
        throw new RuntimeException("invalid");
    }

    String jti = claims.getId();
    Long userId = claims.get("uid", Long.class);

    if (!redis.exists("login:refresh:" + jti)) {
        throw new RuntimeException("expired");
    }

    String newAccess =
        jwtTokenService.createToken(userId, claims.getSubject(), ACCESS);

    Claims newClaims = jwtTokenService.parseToken(newAccess);
    redis.set("login:access:" + newClaims.getId(), userId, 15min);

    return new LoginResult(newAccess, refreshToken);
}
```

# 最后一部分：你现在应该“这样理解自己做的事”

- JWT：身份证
- Redis：门禁系统
- Access：临时通行证
- Refresh：长期通行证
- Filter：门卫

**门卫只看通行证，不看档案**
