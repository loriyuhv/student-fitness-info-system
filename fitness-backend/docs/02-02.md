# 一、目标

> **JWT + Redis 在你这个系统里的职责只有三件事：**

1️⃣ **JWT：证明“你是谁”**
2️⃣ **Redis：保存“你能干什么 + Token 状态”**
3️⃣ **Spring Security Filter：把两者接进安全链**

⚠️ 核心原则（你前面已经选对了）：

- ❌ **权限不放 JWT**
- ❌ **角色不放 JWT**
- ✅ JWT 只放：`userId`、`username`、`tokenType`
- ✅ 权限、角色 → Redis

# 三、Token 设计（这是整套方案的“定盘星”）

## 1️⃣ 两种 Token（你之前说过的）

| Token         | 用途        | 存储                     |
| ------------- | ----------- | ------------------------ |
| Access Token  | 请求鉴权    | Header                   |
| Refresh Token | 换新 Access | HttpOnly Cookie / Header |

### 统一约定

```text
AccessToken  ：15 分钟
RefreshToken：7 天
```

------

## 2️⃣ JWT Payload 设计（极简且安全）

### Access Token Payload

```json
{
  "userId": 10001,
  "username": "20231234",
  "type": "ACCESS",
  "iat": 1700000000,
  "exp": 1700000900
}
```

### Refresh Token Payload

```json
{
  "userId": 10001,
  "type": "REFRESH",
  "iat": 1700000000,
  "exp": 1700600000
}
```

❗ **没有权限、没有角色、没有状态**

# 四、基础设施层：JWT 工具（infrastructure.jwt）

```java
@Component
public class JwtTokenProvider {

    private final String secretKey = "fitness-secret-key";
    private final long accessExpire = 15 * 60 * 1000L;
    private final long refreshExpire = 7 * 24 * 60 * 60 * 1000L;

    public String createAccessToken(Long userId, String username) {
        return Jwts.builder()
            .claim("userId", userId)
            .claim("username", username)
            .claim("type", "ACCESS")
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + accessExpire))
            .signWith(Keys.hmacShaKeyFor(secretKey.getBytes()))
            .compact();
    }

    public String createRefreshToken(Long userId) {
        return Jwts.builder()
            .claim("userId", userId)
            .claim("type", "REFRESH")
            .setIssuedAt(new Date())
            .setExpiration(new Date(System.currentTimeMillis() + refreshExpire))
            .signWith(Keys.hmacShaKeyFor(secretKey.getBytes()))
            .compact();
    }

    public Claims parse(String token) {
        return Jwts.parserBuilder()
            .setSigningKey(secretKey.getBytes())
            .build()
            .parseClaimsJws(token)
            .getBody();
    }
}
```

# 五、Redis 设计（这一步你之前最容易混）

## 1️⃣ Redis Key 规范（非常重要）

```text
LOGIN:TOKEN:ACCESS:{userId}   -> accessToken
LOGIN:TOKEN:REFRESH:{userId}  -> refreshToken
LOGIN:PERM:{userId}           -> Set<String>
```

------

## 2️⃣ 登录成功时存什么？

```java
// Access Token 状态
redis.set("LOGIN:TOKEN:ACCESS:" + userId, accessToken, 15min);

// Refresh Token 状态
redis.set("LOGIN:TOKEN:REFRESH:" + userId, refreshToken, 7days);

// 权限集合
redis.set("LOGIN:PERM:" + userId, permissionSet);
```

👉 **以后所有鉴权，不再查数据库**

# 六、应用层：登录用例（application.auth）

这是 **DDD 的精华**：
Controller 不写逻辑，Security 不写业务。

```java
@Service
public class AuthApplicationService {

    @Autowired JwtTokenProvider jwtProvider;
    @Autowired RedisTemplate<String, Object> redis;
    @Autowired CustomUserDetailsService userDetailsService;

    public LoginResult login(String username, String password) {

        SecurityUser user = (SecurityUser)
            userDetailsService.loadUserByUsername(username);

        // 1. 校验密码
        if (!passwordEncoder.matches(password, user.getPassword())) {
            throw new BadCredentialsException("密码错误");
        }

        // 2. 生成 Token
        String accessToken =
            jwtProvider.createAccessToken(user.getUserId(), user.getUsername());
        String refreshToken =
            jwtProvider.createRefreshToken(user.getUserId());

        // 3. 存 Redis
        redis.opsForValue().set(
            "LOGIN:TOKEN:ACCESS:" + user.getUserId(),
            accessToken,
            15, TimeUnit.MINUTES
        );

        redis.opsForValue().set(
            "LOGIN:TOKEN:REFRESH:" + user.getUserId(),
            refreshToken,
            7, TimeUnit.DAYS
        );

        redis.opsForValue().set(
            "LOGIN:PERM:" + user.getUserId(),
            user.getAuthorities()
                .stream()
                .map(GrantedAuthority::getAuthority)
                .collect(Collectors.toSet())
        );

        return new LoginResult(accessToken, refreshToken);
    }
}
```

# 八、Security Filter：JWT 鉴权（infrastructure.security）

这是 **替代 Session 的关键**。

```java
@Component
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    @Autowired JwtTokenProvider jwtProvider;
    @Autowired RedisTemplate<String, Object> redis;

    @Override
    protected void doFilterInternal(
        HttpServletRequest request,
        HttpServletResponse response,
        FilterChain filterChain
    ) throws IOException, ServletException {

        String token = request.getHeader("Authorization");
        if (token != null && token.startsWith("Bearer ")) {

            Claims claims = jwtProvider.parse(token.substring(7));
            Long userId = claims.get("userId", Long.class);

            // 校验 Redis 中 Token 状态
            String redisToken = (String)
                redis.opsForValue().get("LOGIN:TOKEN:ACCESS:" + userId);

            if (token.substring(7).equals(redisToken)) {

                Set<String> perms = (Set<String>)
                    redis.opsForValue().get("LOGIN:PERM:" + userId);

                Set<GrantedAuthority> authorities =
                    perms.stream()
                        .map(SimpleGrantedAuthority::new)
                        .collect(Collectors.toSet());

                UsernamePasswordAuthenticationToken auth =
                    new UsernamePasswordAuthenticationToken(
                        userId, null, authorities
                    );

                SecurityContextHolder.getContext().setAuthentication(auth);
            }
        }

        filterChain.doFilter(request, response);
    }
}
```
